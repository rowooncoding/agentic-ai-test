## 슬래시 커맨드

- `/push` — README 업데이트 → 커밋 → 푸시

**사용 예시**: `/push` → 현재 변경 사항을 정리해 README에 반영하고 푸시

---

## 실행 절차

아래 순서를 반드시 따른다.

### 1. 변경 사항 파악

```bash
git status
git diff
```

- 어떤 파일이 변경됐는지, 어떤 작업이 이루어졌는지 파악한다.

### 2. README.md 업데이트

- `README.md`를 읽고 현재 작업 내용을 반영한다.
- 신규 스킬 추가, 설정 변경, 구조 변경 등이 있으면 해당 섹션을 업데이트한다.
- 없는 섹션이면 적절한 위치에 새로 추가한다.
- **과거 이력을 삭제하지 않는다.** 기존 내용은 유지하고 추가/수정만 한다.

### 3. 커밋

- 변경된 파일을 명시적으로 스테이징한다. (`git add -A` 사용 금지)
- 커밋 메시지는 `feat:` / `fix:` / `docs:` / `refactor:` 접두사를 사용한다.
- 작업 내용을 한국어로 간결하게 작성한다.

```bash
git add <변경된 파일들>
git commit -m "feat: ..."
```

### 4. 푸시

```bash
git push
```
